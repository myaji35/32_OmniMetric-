## 1. 사용자 시나리오 (User Scenarios)

### 1.1 갑(OmniMetric 관리자) 시나리오

#### US-A1: 고객사 온보딩
> "신규 고객사(을)를 등록하고 API Key를 발행하여 데이터 연동을 시작한다."

| 단계 | 사용자 행동 | 시스템 응답 |
|------|-----------|-----------|
| 1 | 커넥터 관리 페이지(`/connectors`) 진입 | 기존 커넥터 목록 카드 표시 |
| 2 | "새 커넥터 추가" 버튼 클릭 | 등록 폼(고객사명, 연동 URL, 인증 방식) 표시 |
| 3 | 정보 입력 후 등록 | API Key 자동 생성 + 표시 (1회 노출) |
| 4 | 키 검증 버튼 클릭 | 을 시스템과 연결 테스트 → 성공/실패 표시 |
| 5 | 스키마 탐색 클릭 | 을 데이터 테이블/컬럼 구조 트리 표시 |

#### US-A2: 분석 모니터링
> "실행 중인 분석 작업의 진행 상태를 확인하고, 완료된 분석의 정확도를 검토한다."

| 단계 | 사용자 행동 | 시스템 응답 |
|------|-----------|-----------|
| 1 | 대시보드(`/`) 진입 | 시스템 상태, 최근 분석 목록 표시 |
| 2 | 진행 중 작업 카드 확인 | 진행률(%) + 현재 단계 표시 |
| 3 | 완료된 분석 클릭 | 리포트 페이지로 이동 |

#### US-A3: 오차 임계값 관리
> "분석 모델의 자가 보정 임계값을 조정하여 자동 재학습 기준을 설정한다."

| 단계 | 사용자 행동 | 시스템 응답 |
|------|-----------|-----------|
| 1 | 시스템 설정 접근 | 현재 임계값 표시 |
| 2 | 임계값 수정 | `PATCH /v1/threshold` 호출 → 확인 |

### 1.2 을(고객사 사용자) 시나리오

#### US-B1: 데이터 업로드 및 현황 파악
> "CSV 데이터를 업로드하고, 데이터 특성을 파악한 뒤 적절한 분석 유형을 추천받는다."

| 단계 | 사용자 행동 | 시스템 응답 |
|------|-----------|-----------|
| 1 | 데이터 페이지(`/data`) 진입 | 업로드 영역 표시 |
| 2 | CSV 파일 드래그 앤 드롭 | 미리보기 테이블 (상위 10행) 표시 |
| 3 | "데이터 분석" 클릭 | WhatDataAI: 컬럼 타입, 결측치, 추천 분석유형 표시 |
| 4 | "EDA 실행" 클릭 | 분포 히스토그램, 상관행렬 히트맵, 이상치 차트 표시 |

#### US-B2: 분석 실행 및 결과 확인 (핵심 시나리오)
> "155종 알고리즘 토너먼트를 실행하고, 왜 특정 알고리즘이 승자인지 근거를 확인한다."

| 단계 | 사용자 행동 | 시스템 응답 |
|------|-----------|-----------|
| 1 | 분석 페이지(`/analyze`) 진입 | 타겟(Y) 변수 선택 드롭다운 표시 |
| 2 | Y 변수 + 분석유형 선택 후 실행 | 작업 ID 반환 + 진행률 폴링 시작 |
| 3 | 진행률 100% 도달 | 리포트 페이지(`/report/{task_id}`)로 자동 이동 |
| 4 | **승자 알고리즘 카드** 확인 | 알고리즘명, R², Adj.R², P-value 표시 |
| 5 | **수학적 공식** 확인 | Y = w₁X₁ + w₂X₂ + ... + b 수식 렌더링 |
| 6 | **변수 기여도 차트** 확인 | 변수별 영향도(%) 바 차트 + 방향(+/-) |
| 7 | **토너먼트 랭킹** 확인 | 전체 알고리즘 R² 순위 테이블 |
| 8 | **XAI 해석** 확인 | SHAP 변수 중요도 차트 |
| 9 | **자연어 리포트** 확인 | 한국어 요약 ("매출에 가장 큰 영향을 미치는 변수는...") |

#### US-B3: What-if 시뮬레이션
> "분석 결과를 바탕으로 변수를 조절하여 예측값 변화를 확인한다."

| 단계 | 사용자 행동 | 시스템 응답 |
|------|-----------|-----------|
| 1 | 시뮬레이션 페이지(`/simulate/{task_id}`) 진입 | 변수별 슬라이더 표시 (현재값 기준) |
| 2 | X₁ 슬라이더를 +20% 조절 | Y 예측값 실시간 업데이트 |
| 3 | "시나리오 저장" 클릭 | 시나리오 비교 테이블에 추가 |
| 4 | 복수 시나리오 비교 | 시나리오별 Y값 비교 테이블 표시 |

#### US-B4: 행동 시나리오 활용
> "분석 결과에서 도출된 IF-THEN 행동 규칙을 확인하고 외부 시스템에 전달한다."

| 단계 | 사용자 행동 | 시스템 응답 |
|------|-----------|-----------|
| 1 | 행동 시나리오 페이지(`/actions/{task_id}`) 진입 | IF-THEN 카드 목록 (우선순위순) |
| 2 | 카드 상세 보기 | 조건(IF) + 행동(THEN) + 예상 효과 표시 |
| 3 | Webhook 설정 | 외부 시스템 URL 입력 → 시나리오 자동 전달 |

#### US-B5: AI 기반 질의응답
> "분석 결과에 대해 자연어로 질문하고 답변을 받는다."

| 단계 | 사용자 행동 | 시스템 응답 |
|------|-----------|-----------|
| 1 | Q&A 페이지(`/qa/{task_id}`) 진입 | 추천 질문 목록 + 채팅 입력창 표시 |
| 2 | "매출에 가장 큰 영향을 미치는 변수는?" 입력 | AI 응답 (변수명 + 기여도 + 근거) |
| 3 | 후속 질문 | 대화 컨텍스트 유지한 답변 |

## 2. 기능 요구사항 (Functional Requirements)

### 2.1 P0 — 필수 (MVP)

| ID | 기능 | 설명 | 관련 API | 검증 기준 |
|----|------|------|---------|----------|
| FR-01 | B2B 커넥터 관리 | 고객사 등록, API Key 발행/갱신/폐기/검증 | `POST/GET/DELETE /v1/connectors` | API Key 발행 후 검증 성공 |
| FR-02 | 분석 결과 & 근거 시각화 | R², 수학적 공식, 변수 기여도, XAI, 토너먼트 랭킹 | `GET /v1/report/{id}` | 5개 섹션 모두 렌더링 |
| FR-03 | 데이터 업로드 + EDA | CSV 업로드, WhatDataAI, 분포/상관/이상치 차트 | `POST /v1/upload`, `/v1/whatdata`, `/v1/eda` | 차트 3종 정상 표시 |
| FR-06 | 분석 실행 + 진행률 | 알고리즘 토너먼트 시작, 진행률 폴링, 완료 후 이동 | `POST /v1/analyze`, `GET /v1/status/{id}` | 진행률 0→100% 표시 |

### 2.2 P1 — 중요

| ID | 기능 | 설명 | 관련 API | 검증 기준 |
|----|------|------|---------|----------|
| FR-04 | What-if 시뮬레이션 | 변수 슬라이더로 예측값 조절, 시나리오 비교 | `POST /v1/simulate` | 슬라이더 조절 시 Y값 변화 |
| FR-05 | 행동 시나리오 | IF-THEN 카드 UI, 우선순위 정렬, Webhook 전달 | `POST /v1/actions/{id}` | 카드 렌더링 + Webhook 설정 |
| FR-08 | 대시보드 (홈) | 시스템 상태, 최근 분석, 빠른 실행 버튼 | `GET /health`, `/v1/connectors` | 3개 섹션 표시 |

### 2.3 P2 — 부가

| ID | 기능 | 설명 | 관련 API | 검증 기준 |
|----|------|------|---------|----------|
| FR-07 | AI Q&A 채팅 | 분석 결과 기반 자연어 질의응답, 추천 질문 | `POST /v1/qa/{id}` | 질문 입력 → 답변 표시 |

## 3. 비기능 요구사항 (Non-Functional Requirements)

| ID | 카테고리 | 요구사항 | 측정 기준 |
|----|---------|---------|----------|
| NFR-01 | 성능 | 차트 렌더링 1초 이내 | Lighthouse Performance Score ≥ 80 |
| NFR-02 | 사용성 | 비전문가도 15분 내 첫 분석 완료 | 사용자 테스트 기준 |
| NFR-03 | 디자인 | SLDS 스타일 가이드 준수 | Feather Icons, Card 기반, 3-Column |
| NFR-04 | 접근성 | 색상 대비 WCAG AA 기준 충족 | Contrast ratio ≥ 4.5:1 |
| NFR-05 | 안정성 | API 서버 다운 시 graceful fallback | 에러 메시지 + 재시도 버튼 표시 |

## 4. API 연동 매트릭스

| API 엔드포인트 | 메서드 | 사용 페이지 | FR |
|---------------|--------|-----------|-----|
| `/health` | GET | 대시보드 | FR-08 |
| `/v1/connectors` | GET/POST | 커넥터 관리 | FR-01 |
| `/v1/connectors/{id}` | GET/DELETE | 커넥터 상세 | FR-01 |
| `/v1/connectors/{id}/renew` | POST | 키 갱신 | FR-01 |
| `/v1/connectors/{id}/verify` | POST | 키 검증 | FR-01 |
| `/v1/connectors/{id}/schema` | GET | 스키마 탐색 | FR-01 |
| `/v1/connectors/{id}/sync` | POST | 데이터 수집 | FR-01 |
| `/v1/upload` | POST | 데이터 업로드 | FR-03 |
| `/v1/whatdata` | POST | WhatDataAI | FR-03 |
| `/v1/eda` | POST | EDA | FR-03 |
| `/v1/analyze` | POST | 분석 실행 | FR-06 |
| `/v1/status/{task_id}` | GET | 진행률 조회 | FR-06 |
| `/v1/report/{task_id}` | GET | 분석 결과 | FR-02 |
| `/v1/simulate` | POST | 시뮬레이션 | FR-04 |
| `/v1/actions/{task_id}` | POST | 행동 시나리오 | FR-05 |
| `/v1/actions/{task_id}/history` | GET | 시나리오 이력 | FR-05 |
| `/v1/qa/{task_id}` | POST | AI Q&A | FR-07 |
| `/v1/threshold` | PATCH | 임계값 설정 | - |
