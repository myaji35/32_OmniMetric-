## 1. 프로젝트 개요 (Overview)

OmniMetric은 데이터의 '의미 있는 변수'를 찾기 위해 60여 가지 통계 알고리즘을 병렬로 가동하여 최적의 결정계수($R^2$)를 도출하고, 그 근거를 수학적 공식과 자연어 리포트로 제공하는 고성능 백엔드 분석 엔진입니다.

### 1.1 핵심 미션

- **통찰의 자동화**: 데이터 과학자 없이도 데이터 내 핵심 동인(Key Driver)을 즉시 판별.
- **신뢰의 수치화**: $R^2$ 기반의 최적 모델 선정으로 의사결정의 객관적 근거 확보.
- **설명 가능한 AI (XAI)**: "왜?"라는 질문에 통계적 근거와 자연어 리포트로 응답.

---

## 2. 주요 기능 요구사항 (Core Features)

### 2.1 60+ 알고리즘 토너먼트 엔진

- **전수 조사**: 선형/비선형 회귀, 트리 기반 모델, 정보 이론 등 60개 이상의 알고리즘을 입력 데이터에 무차별 대입(Brute-force).
- **승자 독식**: 각 모델의 $R^2$, Adj. $R^2$, P-value를 비교하여 가장 설명력이 높은 최적의 모델을 자동 선정.

### 2.2 초고속 병렬 처리 시스템 (Parallel Engine)

- **멀티프로세싱 최적화**: 60여 개의 알고리즘을 각각 독립된 워커(Worker)에 할당하여 동시 실행.
- **API 비동기 응답**: 대규모 데이터셋 분석 시 작업 ID를 즉시 반환하고, 완료 시 Webhook 또는 폴링으로 결과 전달.

### 2.3 수학적 공식 및 변수 기여도 도출

- **공식 변환**: 선정된 모델을 외부 시스템에서 재사용 가능한 수학 공식($Y = w_1X_1 + w_2X_2 + ... + b$)으로 출력.
- **영향력 분석**: 각 컬럼 변수가 결과값에 미치는 가중치를 백분율 및 방향성(+, -)으로 수치화.

### 2.4 Omni-Insight 자동 리포팅

- **자연어 생성 (NLG)**: 통계적 수치를 "A 변수가 10% 증가할 때 B가 5% 상승함"과 같은 인간의 언어로 변환.
- **도출 근거 제시**: 사용된 알고리즘 리스트와 선정 사유를 포함한 '분석 근거 보고서' 생성.

### 2.5 자가 보정 MLOps (Auto-Retraining)

- **오차 감지**: 실제 운영 데이터와 예측 공식의 오차가 설정 임계값(Threshold)을 벗어날 경우 자동 감지.
- **재학습 파이프라인**: 오차 발생 시 60개 알고리즘 토너먼트를 재개최하여 공식을 실시간 보정.

## 3. 기술 아키텍처 및 스택 (Technical Stack)

- **Backend**: Python 3.11 (FastAPI) - 비동기 I/O 및 고성능 API 처리.
- **Parallelism**: Ray 또는 Celery (Redis 기반) - 60+ 알고리즘 병렬 분산 처리.
- **Analysis**: PyCaret, Scikit-learn, Statsmodels - 통계 및 ML 라이브러리.
- **XAI**: SHAP, LIME - 모델 해석 엔진.
- **Interface**: RESTful API (JSON 포맷 응답).

---

## 4. 데이터 플로우 (Data Flow)

1. **입력**: 외부 시스템(누리팜, 인슈어그래프 등)이 분석용 CSV/JSON 데이터를 API로 전송.
2. **분산**: OmniMetric 메인 엔진이 데이터를 60개 알고리즘 워커로 병렬 배분.
3. **경쟁**: 각 워커가 독립적으로 $R^2$를 계산하고 결과를 중앙으로 집계.
4. **선정**: 최고 성능 모델을 확정하고 수학 공식 및 XAI 통찰 추출.
5. **출력**: 분석 수치, 공식, 자연어 리포트가 담긴 최종 JSON 반환.
6. **감시**: 운영 중 오차가 발생하면 자가 보정 로직 트리거.

## 5. API 규격 요약 (API Spec Summary)

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/v1/analyze` | 데이터 분석 및 알고리즘 토너먼트 시작 |
| GET | `/v1/report/{task_id}` | 생성된 자연어 리포트 및 통계 결과 조회 |
| PATCH | `/v1/threshold` | 오차 감지 및 재학습 임계값 설정 변경 |
